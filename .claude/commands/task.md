Task Analysis

Analyze the current task carefully before writing any code.

Read and validate against:
- docs/business-rules.md
- docs/security.md
- relevant workflow docs

Objectives
- Summarize the requirement
- Identify backend/frontend impact
- Inspect related files and patterns
- Identify risks and edge cases
- Create a minimal implementation plan

Requirements
- Follow CLAUDE.md workflow and conventions
- Reuse existing architecture and patterns
- Keep implementation minimal and consistent
- Do NOT start coding immediately for large changes

Output Format

## Summary
Short explanation of the task.

## Affected Files
List likely files/modules involved.

## Risks
Potential edge cases or breaking changes.

## Plan
Step-by-step implementation approach.
